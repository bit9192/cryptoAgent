# cli/ex:lon 模块开发计划

目标：为 ex:lon 提供上下分区交互界面（上方结果展示、下方命令输入），并支持条件输入面板（如私钥配置）；流程严格遵循先样本后实现。

## 1. 需求范围

本模块当前需求：

1. 上下分区 UI 骨架
- 上半区展示最近任务结果与状态
- 下半区保留命令输入提示符

2. 条件输入面板
- 当任务请求交互输入时，显示输入提示区
- 支持 password/confirm/select/text

3. 安全交互约束
- 密码与私钥类输入不回显
- 敏感值不进入命令历史

4. 与执行链路解耦
- CLI 只负责展示与输入采集
- 不在 CLI 实现业务状态机

测试数据来源：

- 使用本目录 cli-test-data.md
- 遵循 src/test/TEST_DATA_RULES.md

## 2. 分层落位

1. cli
- ex:lon 展示与输入交互
- 交互面板状态的最小渲染函数

2. execute
- 任务状态管理与恢复逻辑（仅消费 CLI 输入）

3. modules/context-engine
- 上下文读写与字段分层（CLI 只读）

## 3. 统一开发流程

1. 先补需求说明
2. 先补测试样本（happy/edge/invalid/security）
3. 先写切片计划
4. 再开发实现
5. 先跑切片测试，再跑受影响测试

## 4. 当前切片计划

### Slice CLI-1：上下分区骨架 + 条件输入状态钩子

本次只做：

1. 在 ex:lon 增加“上方结果面板”文本渲染骨架
2. 下方继续使用现有 prompt 输入，不改变命令语法
3. 交互开始/结束时记录并显示 `awaiting_input` 状态
4. 保持现有 task 执行语义与输出结构不变

本次不做：

1. 全屏 TUI / curses 布局
2. 多任务并行面板
3. 中断恢复后的面板还原
4. 可视化表单组件

验收标准：

1. 执行 task 后可看到上方结果摘要
2. 进入 interact 时面板状态切为 awaiting_input，完成后恢复 idle/running
3. 现有 lon 命令行为不回归

### Slice CLI-2：条件输入面板字段摘要规范

本次只做：

1. awaiting_input 状态展示交互字段摘要（name/type/required）
2. 密码类字段仅展示类型，不展示默认值或敏感内容
3. 交互结束后面板状态恢复并保留最近任务上下文
4. 保持现有命令语义与执行路径不变

本次不做：

1. 全屏表单组件
2. 多任务并行交互面板
3. checkpoint 恢复后的 UI 还原

验收标准：

1. task 请求交互时，面板可展示字段摘要
2. password 字段不会泄露值或敏感提示内容
3. 旧测试不回归
